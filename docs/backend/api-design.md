# API 设计

## RESTful API

### 设计原则
- 资源导向（名词复数）
- HTTP 方法语义化
- 状态码正确使用

### 最佳实践
- 版本管理（`/api/v1/`）
- 分页、过滤、排序
- 错误响应格式统一

## 认证与授权

- JWT：无状态认证
- OAuth 2.0：第三方授权
- Session vs Token

## GraphQL

- Schema 定义
- Query、Mutation、Subscription
- 与 REST 的对比

## API 安全

- 限流（Rate Limiting）
- CORS 配置
- 输入验证
