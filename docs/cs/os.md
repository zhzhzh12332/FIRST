# 操作系统

操作系统是管理计算机硬件和软件资源的系统软件，为应用程序提供服务。

## 进程管理

### 进程 vs 线程

| 维度 | 进程 | 线程 |
|------|------|------|
| 资源 | 独立地址空间（资源隔离好） | 共享进程地址空间 |
| 切换开销 | 大（需切换页表等） | 小（只需切换寄存器） |
| 通信 | IPC（管道、消息队列、共享内存） | 直接读写共享变量 |
| 稳定性 | 一个进程崩溃不影响其他 | 一个线程崩溃可能影响整个进程 |

### 进程状态
```
创建 → 就绪 ←→ 运行 → 终止
         ↓         ↑
        阻塞 ←────┘
```

### 调度算法

| 算法 | 特点 | 评价 |
|------|------|------|
| FCFS (先来先服务) | 非抢占，按到达顺序 | 平均等待时间长 |
| SJF (最短作业优先) | 选择最短 CPU 爆发的进程 | 可能导致长任务饥饿 |
| RR (时间片轮转) | 每个进程获得固定时间片 | 时间片大小是关键参数 |
| 多级反馈队列 | 多个队列，优先级递减 | **现代 OS 常用** |

::: tip
现代操作系统（Linux、Windows）大多使用 **多级反馈队列（MLFQ）**，结合了多种算法的优点。时间片小的任务自动进入高优先级。
:::

### 进程间通信 (IPC)

```text
方式               数据传输量         速度     典型场景
管道 (Pipe)        小                快       父子进程
消息队列           小                中等     任意进程
共享内存           大                最快     需要高频交换数据
信号量             传递信号          快       同步控制
Socket             任意              慢       跨网络通信
```

## 内存管理

### 虚拟内存
每个进程拥有独立的虚拟地址空间，通过 MMU 映射到物理内存。

**优点：**
- 进程隔离，互不影响
- 可以使用比物理内存更大的空间（Swap 交换区）
- 简化内存分配（连续虚拟地址 + 不连续物理地址）

### 页面置换算法
| 算法 | 思路 | 评价 |
|------|------|------|
| FIFO | 淘汰最早进入的页 | 实现简单，但可能淘汰常用页 |
| LRU | 淘汰最近最少使用的页 | 性能好，开销较高 |
| LFU | 淘汰使用频率最低的页 | 考虑使用频率，但可能"留念"旧数据 |
| Clock | 近似 LRU，使用标志位 | 实践中常用，平衡性能和开销 |

```c
// 页面置换 Clock 算法伪代码
void clock_algorithm(PageTable* pt, int page_id) {
    while (1) {
        if (current->referenced == 0) {
            // 替换此页面
            replace_page(pt, current, page_id);
            return;
        } else {
            current->referenced = 0;  // 清除引用位
            current = current->next;   // 移动指针
        }
    }
}
```

## 并发

### 并发 vs 并行
- **并发**：宏观上同时运行，微观上交替执行（单核）
- **并行**：真正的同时运行（多核）

### 四大同步问题
| 问题 | 描述 | 解决方案 |
|------|------|---------|
| 竞争条件 | 多个进程/线程同时操作共享数据 | 互斥锁 |
| 死锁 | 每个进程都在等待对方释放资源 | 银行家算法、资源有序分配 |
| 饥饿 | 某个进程长期得不到资源 | 公平调度 |
| 活锁 | 进程在不停重试但没有进展 | 随机重试时间 |

### 死锁的四个必要条件
1. **互斥**：资源只能被一个进程占用
2. **占有并等待**：进程持有资源并等待其他资源
3. **不可剥夺**：资源不能被强行拿走
4. **循环等待**：形成等待环路

**破坏任一个即可预防死锁**。实际中最常用的是**资源有序分配法**（破坏循环等待）。

```rust
// Rust 的所有权系统在编译时防止并发问题
use std::sync::{Mutex, Arc};
use std::thread;

fn main() {
    let counter = Arc::new(Mutex::new(0));
    let mut handles = vec![];

    for _ in 0..10 {
        let counter = Arc::clone(&counter);
        handles.push(thread::spawn(move || {
            let mut num = counter.lock().unwrap();
            *num += 1;
        }));
    }

    for handle in handles {
        handle.join().unwrap();
    }
    println!("Result: {}", *counter.lock().unwrap());
}
```

## 文件系统

### 关键概念
- **inode**：存储文件元数据（权限、所有者、大小、数据块指针）
- **目录**：文件名称到 inode 的映射表
- **硬链接 vs 软链接**：硬链接共享 inode，软链接是独立的文件路径

### 文件分配方式
| 方式 | 优点 | 缺点 |
|------|------|------|
| 连续分配 | 读取快 | 产生外部碎片 |
| 链式分配 | 无外部碎片 | 随机访问慢 |
| 索引分配 | 支持直接访问 | 索引块有开销 |
